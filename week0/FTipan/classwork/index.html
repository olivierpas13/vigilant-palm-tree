Internet Protocol Suite

Es un conjunto de protocolos conocidos como TCP/IP, el cual se encuentra dividio en 4 partes:

Aplicación: Define reglas para el intercambio de datos
El HTTP es el estandar el intercambio de datos en la web. Pueden haber varias solicitudes HTTP a partir de una misma conexión.
Transporte: Formas en las que se transportan los paquetes de información
El TCP hace la transferencia de forma segura al verificar la conexión entre el cliente y el servidor mediante SYN/SYN-ACK/ACK.
Network: Acepta y manda paquetes en el internet
Se usa la "IP", que se trata de un identificador único para cada equipo, se clasifican entre públicas y privadas. Actualmente se utiliza mucho la IPv4, pero se trata de pasar a una IPv6.
Data Link: Hardware
Se refiere a la infraestructura física del internet como tal (servidores y cables de conexión).

Internet != WWW

Conexión en la Web

  • URL: Uniform Resource Locator, es la dirección completa hacia una página web en específico.
  • Browser: Conocido como "cliente", es una aplicación que permite interactuar con la información del WWW.
  • Server: Espacio pendiete a la espera de un asolicitud http para devolver una respuesta dentro del contenido de la página web.
  • Dominio: Asocia a una persona u organización mediante un registro único, identificable entre toda la web.
  • DNS: Domain Name System, asigna un nombre a la dirección IP.

Estructura de una página web

HTML
Hypertext Markup Language, define la estructura y distribución de la pagina web
CSS
Cascading Style Sheets Language, permite aplicar estilos a la página web
Java Script
Lenguaje de programación que permite agregar lógica y hacer interactiva a la página web.

DOM (Document Objet Model)

Corresponde a la representación de los elementos que conforman una página web, partiendo desde un documento HTLM.

  • Doctype:Declaración que muestra al navegador que versión de HTML se está usando
  • html:Raiz del documento
  • Head:Se incluyen metadatos, título de la página web, su descripción en el navegador y entre otros. No se muestra esta información al usuario
  • Body:Contiene todo el apartado visual presentado al usuario
